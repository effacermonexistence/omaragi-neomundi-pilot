# Terminology

## OmarAGI

OmarAGI is the main public company and project identity for this repository.

## OmarAGI × NeoMundi technical pilot

The OmarAGI × NeoMundi technical pilot is an early interoperability pilot mapping synthetic NeoMundi runtime observability signals into OmarAGI / REVAS pre-action admissibility artifacts.

This phrase does not imply official partnership, production deployment, customer status, third-party validation, certification, or endorsement.

## RCC — Recursive Collapse Constraint

RCC, Recursive Collapse Constraint, is OmarAGI’s broader routing architecture for preserving claim boundaries, uncertainty, proof requirements, adoption gates, and fallback safety before downstream action.

## REVAS

REVAS is the interface layer used in this pilot to convert behavior observation signals into pre-action admissibility decisions and scored artifacts.

## Pre-action admissibility

Pre-action admissibility is a decision layer that determines whether an observed output can be used, bounded, deferred, escalated, or blocked before any downstream action or claim.

## Runtime observability signal

A runtime observability signal is an observation about a model or system output, such as stability, coherence, semantic variability, factual fragility, risk level, or measurement coverage.

## Governance metrology

Governance metrology means measuring behavior and routing reliability signals in a way that can support auditable governance decisions without overclaiming final truth.

## ControlTower

ControlTower refers to the NeoMundi-side live request/testing surface referenced for a planned focused test. Materials in this repository do not claim that a production ControlTower deployment has occurred.

## Pilot counterpart

NeoMundi is described in this repository only as the pilot counterpart and ControlTower-side signal source for this early interoperability pilot. This does not imply official endorsement, production validation, commercial deployment, or a broader partnership claim.

## Scored artifact

A scored artifact is an OmarAGI-side structured output that records the source observation summary, REVAS route, admissibility decision, confidence/risk estimates, required next step, limitations, and boundary notes.
