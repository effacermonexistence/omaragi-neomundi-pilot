# ControlTower live test plan

Status: planned, not executed.

## Target

A focused ControlTower request batch. The exact request count, task mix, and operating constraints should be confirmed before execution.

## Objective

Test whether NeoMundi runtime observability signal outputs can be repeatedly mapped into OmarAGI / REVAS pre-action admissibility decisions and scored artifacts.

## Candidate request types

- stable explanatory output with hidden factual fragility
- ambiguous reasoning with semantic variability
- bounded reference task with support limitations
- high-stakes explanatory output with contradiction or overclaim
- partial observation or incomplete measurement
- low-risk supported answer
- high-coherence unsupported answer
- uncertainty-heavy answer
- scope boundary violation
- correction after challenge

## Success criteria

A useful live test would show whether the signal-to-decision mapping is repeatable, conservative, auditable, and boundary-preserving. It does not require every observation to be favorable.

## Boundary

No production-readiness, official partnership, customer status, validation, endorsement, certification, cost-reduction, or benchmark-uplift claim should be made from this planned test.
